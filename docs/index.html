Tunnelr.

A self-hosted localhost tunneling tool built by a CRM company that got tired of duct-taping dev workflows.

Why this exists

We build CRMs, automation systems, and AI agents.
That means a lot of webhooks, callbacks, and local services that need to be temporarily public.

Tunnelr started as an internal tool.
We open-sourced it because it solved a real problem for us.

What Tunnelr is (and isn't)

Tunnelr is:

  • Self-hosted
  • Domain-first
  • Simple and predictable

Tunnelr is not:

  • A hosted SaaS
  • A replacement for every tunneling service
  • A "sign up and forget" tool

Where it fits best

  • CRM webhook development
  • AI agent callbacks
  • Internal tools
  • Long-running tunnels without limits
